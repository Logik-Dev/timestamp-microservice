Timestamp Microservice

Cahier des charges:

  1. L'url de l'API est: [url_du_projet]/api/timestamp/:date?
  2. La date fournie est considérée valide si elle peut être convertie par new Date(date).
    Il faut noter aussi que le timestamp unix doit être un entier donné en millisecondes.Dans notre test nous utiliserons une date conforme avec la norme ISO-8601 (ex: "2016-11-20") afin d'obtenir un timestamp UTC.
  3. Si l'utilisateur n'a pas entrer de date, alors on utilisera la date courante avec new Date().
  4. Enfin si la date est invalide la réponse JSON du serveur sera de type:
    {"error": "Invalid Date"}

Exemples d'utilisation:

[url_du_projet]/api/timestamp/2015-12-24
[url_du_projet]/api/timestamp/1450915200000

Exemple de retour:

{"unix": 1450915200000, "utc": "Thu, 24 Dec 2015 00:00:00 GMT"}
